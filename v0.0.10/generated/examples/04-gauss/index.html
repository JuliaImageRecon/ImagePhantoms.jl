Overview

Another useful shape for constructing 2D digital image phantoms is the 2D Gaussian, specified by its center, widths, angle and value. All of the methods in ImagePhantoms support physical units, so we use such units throughout this example. (Using units is recommended but not required.)

Define a 2D Gaussian object, using physical units.

Spectrum using spectrum

There are two ways to examine the spectrum of this image:

  • using the analytical Fourier transform of the 2D Gaussian via spectrum
  • applying the DFT via FFT to the digital image.

Because the shape has units mm, the spectra axes have units cycles/mm.

zscale = 1 / IP.fwhm2spread(1)^2 / prod(width) # normalize spectra by area
spectrum_exact = spectrum(axesf(ig)..., [ob]) * zscale
sp = z -> max(log10(abs(z)/oneunit(abs(z))), -6) # log-scale for display
clim = (-6, 0) # colorbar limit for display
(xlabel, ylabel) = ("Î½â‚", "Î½â‚‚")
p2 = jim(axesf(ig)..., sp.(spectrum_exact), "log10|Spectrum|"; clim, xlabel, ylabel)

Sadly fft cannot handle units currently, so this function is a work-around:

function myfft(x::AbstractArray{<:Any})
    u = unit(eltype(x))
    return fftshift(fft(fftshift(x) / u)) * u
end

spectrum_fft = myfft(img) * dx * dy * zscale
p3 = jim(axesf(ig)..., sp.(spectrum_fft), "log10|DFT|"; clim, xlabel, ylabel)

Compare the DFT and analytical spectra to validate the code

err = maximum(abs, spectrum_exact - spectrum_fft) /
        maximum(abs, spectrum_exact)
@assert err < 1e-6
p4 = jim(axesf(ig)..., 1e6*abs.(spectrum_fft - spectrum_exact), "Difference Ã— 1e6"; xlabel, ylabel)
jim(p1, p4, p2, p3)

Radon transform using radon

Examine the Radon transform of the 2D Gaussian using radon, and validate it using the projection-slice theorem aka Fourier-slice theorem.

dr = 0.02mm # radial sample spacing
nr = 2^10 # radial sinogram bins
r = (-nrÃ·2:nrÃ·2-1) * dr # radial samples
fr = (-nrÃ·2:nrÃ·2-1) / nr / dr # corresponding spectral axis

p5 = jim(r, rad2deg.(Ï•), sino; aspect_ratio=:none, title="sinogram", yflip=false, xlabel="r", ylabel="Ï•")

Note that the maximum sinogram value is about fwhm2spread(5mm) = 5mm * sqrt(Ï€ / log(16)) â‰ˆ 5.3mm which makes sense for a 2D Gaussian whose longest axis has FWHM = 5mm.

The above sampling generated a parallel-beam sinogram, but one could make a fan-beam sinogram by sampling (r, Ï•) appropriately.

The good agreement between the analytical spectra (solid lines) and the DFT samples (disks) validates that phantom, radon, and spectrum are all self consistent for this Gauss2 object.
